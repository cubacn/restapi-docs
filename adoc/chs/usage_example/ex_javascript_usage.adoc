[[rest_api_v2_ex_javascript_usage]]
=== JavaScript 用法示例

本节包含在 JavaScript 中使用 REST API v2 的示例，此处 JavaScript 运行在 HTML 页面中。该页面最初显示登录表单，登录成功后显示消息和实体列表。

为简单起见，我们将使用 `modules/web/web/VAADIN` 文件夹存储 HTML、CSS 和 JavaScript 文件，因为默认情况下，部署的 Web 应用程序的相应的文件夹用来提供静态资源。因此无需对 Tomcat 应用程序服务器进行任何配置。生成的 URL 将以 `++http://localhost:8080/app/VAADIN++` 开头，所以不要在真实环境的应用程序中使用此方法 - 而应该为静态资源创建一个具有特定上下文的单独 Web 应用程序。

下载 https://jquery.com/download[jQuery] 和 http://getbootstrap.com/getting-started/#download[Bootstrap] 并复制到项目的 `modules/web/web/VAADIN` 文件夹。创建 `customers.html` 和 `customers.js` 文件，文件夹的内容应如下所示：

[source, plain]
----
bootstrap.min.css
customers.html
customers.js
jquery-3.1.1.min.js
----

`customers.html` 文件内容：

[source, html]
----
include::{sourcesdir}/rest_customers.html[]
----

`customers.js` 文件内容：

[source, javascript]
----
include::{sourcesdir}/rest_customers_script.js[]
----

来自用户输入的登录名和密码通过 POST 请求发送到服务器，并在 `Authorization` 请求头中使用 Base64 编码的客户端凭证，如<<rest_api_v2_ex_get_token>>部分中所述。如果验证成功，则网页从服务器接收 access token 值，token 存储在 `oauthToken` 变量中，之后，隐藏 `loginForm` div 并显示 `loggedInStatus` div。

需要显示客户列表，发送请求到服务器<<rest_api_v2_ex_get_entities_list,去获取 `sales$Customer` 实体的实例>>，在 `Authorization` 请求头中传递 `oauthToken` 值。

如果成功处理请求，则显示 `customers` div，并且 `customersList` 元素展示包含客户名称和电子邮件的内容。
